(em andamento)

Domain - regra de negocio
deve conter:
- Entidades
- value objects
- regra de negocios puras
- erros de dominio

Repository - interface de acesso a dados
deve conter: apenas interfaces nunca implementações

Service - caso de uso da aplicação
ele orquestra:
 - Entidades do dominio
 - Repositories
 - mensageria
 - regras de fluxo

extensions/adapters - implementações das interfaces
Aqui ficam as "traduções" entre o mundo externo e o dominio
- Implementações dos Repositories(postgres, mongodb, redis)
- client http externos
- mensageria(rabbitmq, kafka)
- cache
- integrações com APIs


gateways/transports - pontos de entrada da aplicação
esta camada recebe requisições, valida e chama o service
- http handlers
- GraphQL
- websocket
